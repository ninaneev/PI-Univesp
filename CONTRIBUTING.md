# Contributing

## Regras do projeto
- Ninguém faz push direto na main
- Toda alteração deve vir por Pull Request
- Cada tarefa deve estar ligada a uma Issue
- Cada branch deve ser criada para uma Issue específica

## Padrão de branches
- feat/issue-<numero>-descricao-curta
- fix/issue-<numero>-descricao-curta
- docs/issue-<numero>-descricao-curta

## Padrão de commits
- feat: ...
- fix: ...
- docs: ...
- chore: ...

## Definição de pronto
Uma Issue só está concluída quando:
- o código foi implementado
- foi testado localmente
- existe Pull Request
- existe evidência (print ou teste no Swagger)
